# Internationalization (i18n)

## Supported Languages

- **Hungarian (hu)** — default
- **English (en)**

## Routing

Locale-prefix routing:
- `/hu/kavek` — Hungarian
- `/en/coffees` — English
- `/` → redirect `/hu`

URL structure (HU → EN):
```
/kavek              → /coffees
/kavek/[slug]       → /coffees/[slug]
/eszkozok           → /equipment
/merch              → /merch
/csomagok           → /bundles
/sztorik            → /stories
/sztorik/[slug]     → /stories/[slug]
/kosar              → /cart
/penztar/...        → /checkout/...
/elofizetes         → /subscription
/belepes            → /login
/regisztracio       → /register
/fiokom/...         → /account/...
```

Admin routes: `/admin/...` (no locale prefix, always Hungarian).

## Language Switcher

In the header, globally accessible:
- Button: "HU" / "EN" toggle (or dropdown)
- On click: navigates to the same page in the other locale
- Language selection persists: in the user profile (logged in), in a cookie (anonymous)

## Content i18n

All content (product names, descriptions, stories) is stored in both languages. The appropriate language version is displayed on the UI based on the current locale.

All user interface text (buttons, labels, error messages) must also be available in both languages.

## Email i18n

All emails are sent according to the user's language preference:
- Logged-in user: profile language
- Anonymous (e.g., gift card recipient): default HU
- Both email subject and body follow the language setting

## Admin Language

The admin interface is Hungarian only. There is no language switcher in the admin section. However, in content editor forms, both language fields are editable (HU tab / EN tab).
